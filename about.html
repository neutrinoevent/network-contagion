Network Contagion
Interactive network simulation • WebGL

Project Overview

This is an interactive WebGL simulation demonstrating how processes spread through networked systems. The same underlying mathematical framework can model epidemics (SIR dynamics), information cascades, systemic failures, or adversarial propagation—depending on how you interpret the state transitions.

This project demonstrates technical fluency across several domains: complex systems modeling, graph theory and network science, real-time WebGL rendering, and interactive data visualization. It's designed to be both visually compelling and algorithmically substantive.

Technical Mechanics

State Machine & SIR Dynamics

Each node operates as a finite state automaton with four states: susceptible (baseline), infected (active propagation), recovered (temporary immunity), and transitions governed by neighbor interactions. The core loop implements a discrete-time stochastic SIR-style model:

  • Infection probability scales with infected neighbor count and edge weights
  • Recovery occurs after a state-dependent dwell time with gaussian noise
  • Temporary immunity prevents immediate reinfection (recovery → susceptible transition)
  • Optional "containment" mode reduces transmission probability (network segmentation / rate limiting)
  • Optional "rewire" mode allows limited topological adaptation (simplified model of evasion or rerouting)

Graph Generation & Topology

The simulation includes six distinct network families, each with different structural properties that dramatically affect spreading dynamics:

  • small_world — Watts-Strogatz model: high clustering + low path length, realistic for social networks
  • scale_free — Barabási-Albert model: power-law degree distribution, vulnerable to targeted hub removal
  • community — Stochastic block model: dense intra-community edges, sparse bridges between groups
  • spatial — Random geometric graph: proximity-based connectivity, models physical/geographic constraints
  • grid — 2D lattice: regular structure, slow but predictable spreading patterns
  • hub_spoke — Star graph with additional chords: extreme centralization, single point of failure

Graphs are generated using networkx with deterministic seeds, then laid out using force-directed spring algorithms (nx.spring_layout). Node centrality (betweenness) is pre-computed to inform visual sizing and strategic importance.

Rendering & Performance

The visualization uses sigma.js (WebGL renderer) with graphology for graph data structures. This allows smooth 60 FPS rendering of networks with 200–300 nodes. State changes trigger color interpolation and edge "pulse" animations to visualize transmission events in real-time. The renderer handles pan, zoom, and node interaction without performance degradation.

Why this matters: Many network simulations use toy graphs or abstract visualizations. This project uses realistic network topologies, implements actual epidemic models, and renders everything interactively in the browser. It's a full-stack demonstration: Python graph generation → JSON data → JavaScript simulation engine → GPU-accelerated rendering.

Controls & Interaction

  • Tap a node — Inject infection at that specific node (localized seeding)
  • Inject — Inject at random node (stochastic seeding)
  • Contain — Reduce transmission probability across all edges (intervention mode)
  • Rewire — Allow limited adaptive edge rewiring (toy model of network adaptation)
  • Reset — Return all nodes to susceptible state
  • Speed — Pinch gesture (mobile) or Ctrl+wheel (desktop) to adjust simulation rate

The visualization updates in discrete time steps, but animation interpolation creates the illusion of continuous state evolution. Edge pulses fade over several frames to show directional transmission pathways.

Visual Design

  • Blue — Susceptible / baseline state
  • Pink → Amber — Infected (fresh → older infections, color shift over time)
  • Green — Recovered / temporarily immune
  • Edges briefly "pulse" when transmission occurs (white glow along edge path)

Node sizes scale with betweenness centrality—more central nodes are visually larger, making structural importance immediately apparent. Color transitions use smooth interpolation rather than hard state changes.

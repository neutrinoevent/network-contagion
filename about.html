Network Contagion
Interactive network simulation • WebGL

What this is

This is a small interactive demo that visualizes how “something” can spread across a network. Depending on the mode, you can interpret it as an epidemic, information diffusion, cascading failure, or even attacker / defender dynamics in a networked system.

The goal here isn’t to claim a perfect real-world model—it's to show clear systems thinking, readable visuals, and a working simulation that’s easy to extend.

Graph theory Complex systems Simulation WebGL visualization

The basic idea (plain language)

  • Nodes are agents (people, devices, accounts, sensors, etc.).
  • Edges are pathways (contact, connectivity, communication, dependence).
  • Each node changes state over time based on neighbors and the selected mode.

Under the hood it’s a lightweight state machine inspired by SIR-style contagion models: susceptible → infected → recovered (with temporary resistance).

How to read the visualization

  • Blue: susceptible / baseline
  • Pink → Amber: infected (fresh → older)
  • Green: recovered / temporarily resistant
  • Edges briefly “pulse” when a transmission event occurs (a visible pathway trace).

Controls

  • Tap a node: inject at that node (start a local event)
  • Inject: inject at a random node
  • Contain: reduces spread (think throttling, segmentation, quarantine)
  • Rewire: allows limited adaptive connection changes (toy model of rerouting/evasion)
  • Reset: return to baseline state
  • Speed: pinch (mobile) / ctrl+wheel (desktop) adjusts simulation speed

Why multiple network variants

The same rules can look totally different depending on topology: grid-like networks, small-world networks, and scale-free networks produce different cascade patterns. This demo includes multiple generated variants so it doesn’t feel “canned.”
